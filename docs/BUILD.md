# Build Guide — JS Tap-Tap

This guide explains how to build the standalone `JS Tap-Tap.exe` from source
using Python and PyInstaller.

> **End users do not need this.** Download the pre-built EXE from the
> [Releases page](../../../releases/latest).

---

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Python | 3.11 or 3.12 (64-bit) | [python.org](https://www.python.org/downloads/) |
| pip | bundled with Python | — |
| Git | any | [git-scm.com](https://git-scm.com/) |

---

## Step 1 — Clone the Repository

```bash
git clone https://github.com/yourusername/js-tap-tap.git
cd js-tap-tap
```

---

## Step 2 — Create a Virtual Environment

```powershell
# Windows PowerShell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

---

## Step 3 — Install Dependencies

```bash
pip install -r requirements.txt
```

This installs `pygame`, `pillow`, and `pyinstaller` at the pinned versions.

---

## Step 4 — Verify the Game Runs from Source

Before building, confirm the game works correctly from source:

```bash
python -m src.js_tap_tap.main
```

The game window should open and behave identically to the released EXE.
If it does not start, check that your Python version and `pygame` install are correct.

---

## Step 5 — Build the Standalone EXE

```bash
pyinstaller packaging/game.spec
```

PyInstaller will:
1. Analyse `src/js_tap_tap/main.py` and its imports.
2. Bundle everything into a single `dist/JS Tap-Tap.exe`.
3. Embed `assets/branding/icon.ico` as the EXE icon.

**Output location:** `dist/JS Tap-Tap.exe`

> The `build/` folder generated by PyInstaller is automatically excluded by
> `.gitignore`. **Never commit it.**

---

## Step 6 — Test the Built EXE

Run the generated EXE to confirm it works:

```powershell
.\dist\"JS Tap-Tap.exe"
```

Verify:
- The window opens with the title **"JS Tap-Tap"**.
- Login / Register works.
- Gameplay runs correctly.
- `users.json` is created in the working directory.

---

## Step 7 — Package for Release

For an official release, package the EXE along with the installer:

```powershell
# Create the release ZIP (run from repo root)
$version = Get-Content VERSION
$zip = "JS-Tap-Tap-Setup-v$version.zip"
Compress-Archive -Path `
    "dist\JS Tap-Tap.exe", `
    "installer\Install.bat", `
    "installer\welcome.py", `
    "assets\branding\icon.ico" `
    -DestinationPath $zip

# Generate checksums
Get-FileHash $zip -Algorithm SHA256
Get-FileHash "dist\JS Tap-Tap.exe" -Algorithm SHA256
```

Upload the ZIP and the standalone EXE to the GitHub Release, along with
the checksums in `checksums-vX.Y.Z.txt`.

---

## Updating the Spec File

If you rename entry points or move source files, update `packaging/game.spec`:

- `Analysis([...])` — update the path to `main.py`
- `icon=` — update if the ICO file moves
- `name=` — the output EXE filename (keep as `"JS Tap-Tap"`)

See [PyInstaller docs](https://pyinstaller.org/en/stable/spec-files.html) for
full spec file reference.
